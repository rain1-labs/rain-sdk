# v1.1.0 New Features

## Dispute & Appeal

### `buildCreateDisputeTx`

Builds one or more transactions to open a dispute on a resolved market. If the market's base token requires approval, an approval transaction is included automatically.

A dispute challenges the outcome chosen by the market resolver. The dispute fee is read on-chain from the market contract.

```typescript
const txs = await rain.buildCreateDisputeTx({
  marketId: "664f1a2b...",
  walletAddress: "0xYourWallet...",
  usdtTokenAddress: "0xFd086bC7...",   // required for USDT markets
  rainTokenAddress: "0xRainToken...",   // required for RAIN markets
});

// txs[0] may be an ERC-20 approval (if allowance is insufficient)
// txs[last] is the openDispute call on the market contract
for (const tx of txs) {
  await walletClient.sendTransaction(tx);
}
```

**Parameters (`CreateDisputeTxParams`)**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `marketId` | `string` | Yes | The market's ID (from `getPublicMarkets` or `getMarketById`) |
| `walletAddress` | `` `0x${string}` `` | Yes | The wallet address opening the dispute |
| `usdtTokenAddress` | `` `0x${string}` `` | Conditional | Required if the market uses USDT as its base token |
| `rainTokenAddress` | `` `0x${string}` `` | Conditional | Required if the market uses RAIN as its base token |

**Returns:** `Promise<RawTransaction[]>` — one or two transactions (approval + dispute).

---

### `buildCreateAppealTx`

Builds transactions to appeal a dispute decision. Identical interface to `buildCreateDisputeTx` — appeals escalate the dispute to a higher-level resolution mechanism.

```typescript
const txs = await rain.buildCreateAppealTx({
  marketId: "664f1a2b...",
  walletAddress: "0xYourWallet...",
  usdtTokenAddress: "0xFd086bC7...",
});

for (const tx of txs) {
  await walletClient.sendTransaction(tx);
}
```

**Parameters (`CreateAppealTxParams`)** — same as `CreateDisputeTxParams`.

**Returns:** `Promise<RawTransaction[]>`

---

### `buildExtendTimeTx`

Builds a transaction to extend the dispute/appeal timer on a market. Requires an authenticated session (`accessToken` from `login()`). The SDK fetches a signed timestamp from the Rain backend and targets the market's oracle contract.

```typescript
const tx = await rain.buildExtendTimeTx({
  marketContractAddress: "0xMarketContract...",
  walletAddress: "0xYourWallet...",
  accessToken: "eyJhbGciOi...",
});

await walletClient.sendTransaction(tx);
```

**Parameters (`ExtendTimeTxParams`)**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `marketContractAddress` | `` `0x${string}` `` | Yes | The market's on-chain contract address |
| `walletAddress` | `` `0x${string}` `` | Yes | The wallet requesting the extension |
| `accessToken` | `string` | Yes | JWT from `rain.login()` |

**Returns:** `Promise<RawTransaction>` — a single transaction targeting the oracle contract's `extendTime` function.

## Cancel All Open Orders

### `buildCancelAllOpenOrdersTx`

Fetches all open orders for a wallet on a given market and builds cancel transactions for each one. Groups cancellations by type (buy/sell) and returns the minimal set of transactions needed.

Requires authentication since it queries the user's order book.

```typescript
const txs = await rain.buildCancelAllOpenOrdersTx({
  marketId: "664f1a2b...",
  marketContractAddress: "0xMarketContract...",
  walletAddress: "0xYourWallet...",
  accessToken: "eyJhbGciOi...",
});

if (txs.length === 0) {
  console.log("No open orders to cancel");
} else {
  for (const tx of txs) {
    await walletClient.sendTransaction(tx);
  }
}
```

**Parameters (`CancelAllOpenOrdersTxParams`)**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `marketId` | `string` | Yes | The market's ID |
| `marketContractAddress` | `` `0x${string}` `` | Yes | The market's on-chain contract address |
| `walletAddress` | `` `0x${string}` `` | Yes | The wallet whose orders to cancel |
| `accessToken` | `string` | Yes | JWT from `rain.login()` |

**Returns:** `Promise<RawTransaction[]>` — zero or more cancel transactions. Returns an empty array if no open orders exist.

## Login

### `login`

Authenticates a wallet with the Rain backend using a signed message. Returns a JWT access token and user ID. The access token is required by authenticated methods like `buildCancelAllOpenOrdersTx` and `buildExtendTimeTx`.

The message to sign is the lowercased wallet address, signed via `personal_sign`.

```typescript
// 1. Sign the message
const message = walletAddress.toLowerCase();
const signature = await walletClient.signMessage({ message });

// 2. Authenticate
const { accessToken, userId } = await rain.login({
  signature,
  walletAddress,
  smartWalletAddress: smartAccount, // from RainAA.connect()
});

// 3. Use the token in authenticated calls
const txs = await rain.buildCancelAllOpenOrdersTx({
  marketId: "...",
  marketContractAddress: "0x...",
  walletAddress,
  accessToken,
});
```

**Parameters (`LoginParams`)**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `signature` | `string` | Yes | `personal_sign` of the lowercased wallet address |
| `walletAddress` | `string` | Yes | EOA wallet address |
| `smartWalletAddress` | `string` | Yes | Smart account address (from `RainAA.connect()`) |
| `referredBy` | `string` | No | Optional referral code |

**Returns:** `Promise<LoginResult>`

```typescript
interface LoginResult {
  accessToken: string;  // JWT — pass to authenticated SDK methods
  userId: string;       // Backend user ID
}
```

## RainSocket

### Real-time market events

`RainSocket` provides a WebSocket connection to the Rain backend for live market updates. It wraps `socket.io` and exposes typed event subscriptions. Each subscription returns an unsubscribe function.

```typescript
import { RainSocket } from '@buidlrrr/rain-sdk';

const socket = new RainSocket({ environment: 'production' });

socket.onConnect(() => {
  console.log('Connected to Rain WebSocket');
});
```

**Constructor** — accepts `RainCoreConfig` (same as `Rain`). Only `environment` is used.

### Available Events

| Method | Event | Callback Data |
|---|---|---|
| `onEnterOption(marketId, cb)` | User buys into an option | `EnterOptionEventData` |
| `onOrderCreated(marketId, cb)` | New limit order placed | `OrderEventData` |
| `onOrderCancelled(marketId, cb)` | Order cancelled | `OrderEventData` |
| `onOrderFilled(marketId, cb)` | Order partially/fully filled | `OrderFilledEventData` |
| `onDisputeOpened(marketId, cb)` | Dispute opened on a market | `DisputeOpenedEventData` |
| `onAppealOpened(marketId, cb)` | Appeal filed | `AppealOpenedEventData` |
| `onDisputeTimeExtended(marketId, cb)` | Dispute timer extended | `DisputeTimeExtendedEventData` |
| `onDisputeWinner(marketId, cb)` | Dispute winner decided | `DisputeWinnerEventData` |
| `onAppealWinner(marketId, cb)` | Appeal winner finalized | `AppealWinnerEventData` |
| `onClaimReward(marketId, userId, cb)` | User's claim confirmed | `ClaimRewardEventData` |

### Example: Live trade feed

```typescript
const socket = new RainSocket({ environment: 'production' });
const marketId = "664f1a2b...";

// Subscribe to trades
const unsubscribe = socket.onEnterOption(marketId, (data) => {
  for (const inv of data.investments) {
    console.log(`Option ${inv.optionName}: ${inv.amount} wei invested`);
  }
});

// Subscribe to limit orders
const unsubOrders = socket.onOrderCreated(marketId, (data) => {
  console.log(`New ${data.order.orderType} order at ${data.order.pricePerShare}`);
});

// Later: clean up
unsubscribe();
unsubOrders();
socket.disconnect();
```

### Key Types

```typescript
interface EnterOptionEventData {
  poolId: string;
  investments: {
    choiceIndex: number;
    amount: string;        // raw wei
    optionName?: string;
  }[];
  totalInvestmentWei: string;
  tokenDecimals: number;
}

interface OrderEventData {
  poolId: string;
  order: {
    _id: string;
    orderType: 'buy' | 'sell';
    option: number;
    optionName?: string;
    pricePerShare: string;
    quantity: string;       // raw wei
    createdAt: string;
  };
  tokenDecimals: number;
}

interface DisputeOpenedEventData {
  poolId: string;
  isDisputed: boolean;
  isAiResolver: boolean;
  status: string;
  disputeData: any;
  winnerDecidedBy: string;
}
```

## getUserInvestments

### `getUserInvestments`

Fetches all markets a wallet has invested in, with investment details. Supports pagination and filtering by market status. Requires authentication.

```typescript
const investments = await rain.getUserInvestments({
  walletAddress: "0xYourWallet...",
  accessToken: "eyJhbGciOi...",
  status: "Live",    // optional: filter by market status
  limit: 20,         // optional: pagination
  offset: 0,
});

for (const inv of investments) {
  console.log(`Market ${inv.poolId}: ${inv.totalInvestment} invested`);
}
```

**Parameters (`GetUserInvestmentsParams`)**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `walletAddress` | `string` | Yes | The wallet address to query |
| `accessToken` | `string` | Yes | JWT from `rain.login()` |
| `status` | `MarketStatus` | No | Filter by market status (`'Live'`, `'Closed'`, `'UnderDispute'`, etc.) |
| `limit` | `number` | No | Max results to return |
| `offset` | `number` | No | Number of results to skip (for pagination) |

**Returns:** `Promise<UserInvestment[]>`

```typescript
interface UserInvestment {
  _id: string;
  poolId: string;
  walletAddress: string;
  investment: any[];
  totalInvestment: string;
  pool: Record<string, any>;  // embedded market data
}
```

**`MarketStatus` values:** `'Live'` | `'New'` | `'WaitingForResult'` | `'UnderDispute'` | `'UnderAppeal'` | `'ClosingSoon'` | `'InReview'` | `'InEvaluation'` | `'Closed'` | `'Trading'`

## getMarketById

### `getMarketById`

Fetches a single market by its ID. Useful when you already have a market reference from another API call, a transaction event, or `RainSocket`.

```typescript
const market = await rain.getMarketById({
  marketId: "664f1a2b...",
});

console.log(market.title, market.status);
```

**Parameters (`GetMarketByIdParams`)**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `marketId` | `string` | Yes | The market's ID |

**Returns:** `Promise<Market>`

```typescript
interface Market {
  id: string;
  title: string;
  totalVolume: string;
  status: MarketStatus;
  poolOwnerWalletAddress?: string;
  contractAddress?: string;
}
```
